oram

local-first recursive audio workstation.

"We seem to have tuned circuits within us" — Daphne Oram

record sound → ORAM listens for pitch, tempo, key and spectral shape → the engine router creates a complementary layer through the selected local or BYOK provider. layers stay editable as recorder, looper, sampler and archive material.

workflow

  • record — capture audio from your mic into the selected layer
  • listen — oram analyzes pitch, BPM, key, harmonics, and spectral character
  • generate — AI creates a new layer constrained by the analysis but free in imagination
  • layer — stack up to 4 layers, each aware of the others' spectral space
  • transform — apply DSP effects: reverse, granular, reverb, pitch-shift…

controls

  • ⏺ record from mic into selected layer
  • ⊕ overdub onto selected layer
  • fx open DSP transforms (reverse, granulate, reverb…)
  • ✦ summon — listen to what's sounding and generate a new layer
  • auto auto-generative mode: record → listen → generate automatically
  • ◉ export mix · ⊘ hard-silence capture/layers/pending output · + add layer

layers

  • corners — top-left: select/mute (right-click: solo) · top-right: export · bottom-left: generate from · bottom-right: clear
  • waveform — drag to set loop region · click to clear
  • vol strip — drag, scroll, double-click for unity
  • prompt — type a description below and press ↵

keys

  • 1–4 select · m mute · u unmute all
  • r record · o overdub · k kill · g generate · l listen
  • ⌘K command palette · / focus prompt · esc close

engines

ORAM exposes Local Mock by default and currently routes generation to configured ElevenLabs, Stability AI, fal Stable Audio, or local sidecar engines. Hugging Face and Replicate credentials can be tracked, but their adapters are not active yet. keys stay server-side and are never sent to the dashboard.

oram credentials set elevenlabs
oram credentials set stability
oram credentials status

generation is always non-vocal — oram produces sound textures, effects, and music, never speech.
